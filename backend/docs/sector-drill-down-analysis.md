## 4. 实现方案

### 4.1 数据流

```
AKShare板块资金流排名（已有）
        ↓ 取Top3板块名称
板块代码查询（name → BK代码）
        ↓
成分股获取（BK代码 → 成分股列表）
        ↓
区间涨幅计算（成分股代码 → 历史行情 → 区间涨幅）
        ↓
排序输出 = 板块统计（区间领涨股票）
```

### 4.2 关键设计

1. **板块代码缓存**：`stock_board_industry_name_ths()` 返回的板块名→代码映射缓存在Redis（板块列表变化低频，1天过期即可）
2. **区间涨幅计算**：取成分股列表后，批量调 `stock_zh_a_hist()` 算区间涨幅
3. **频率控制**：每个板块成分股可能50-200只，按1秒间隔批量取历史行情，单次约3-5分钟
4. **接口复用**：沿用现有 `DataFetcher<T>` 抽象，新增 `SectorDetailFetcher` 和 `SectorStatsFetcher`

### 4.3 性能与风控

| 维度 | 评估 |
|------|------|
| API调用量 | Top3板块 × ~100只成分股 × 1次hist = ~300次 |
| 总耗时 | 300次 × 1秒间隔 ≈ 5分钟 |
| 风控风险 | 低频（每天1次）+ 1秒间隔，不触发限流 |
| 数据量 | 每只股约20天日线 ≈ 6000条，全量<2MB |

### 4.4 降级策略

如果成分股数量过多或API限流：
1. **只取领涨股前10**：不需要算全部成分股的区间涨幅
2. **用成分股API自带涨幅排序**：`stock_board_industry_cons_ths()` 返回已含涨跌幅，先按日涨幅排，只对Top10算区间涨幅
3. **缓存历史行情**：同一只股票的历史行情只取一次，多板块共用

## 5. 不能直接覆盖的部分
