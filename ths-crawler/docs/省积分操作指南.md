# 省积分操作指南

## 核心机制

扣子按token计费，**每轮对话的输入 = 完整历史上下文 + 本轮消息**。第1轮对话输入可能500 token，第20轮就膨胀到2万+，越往后每轮成本指数级增长。

**所以：会话长度是积分消耗的最大变量，没有之一。**

---

## 第一梯队：效果最显著（省30-50%）

### 1. 主动换会话

- **时机**：一个任务做完了、对话"重"了（超过15-20轮）、或你要切换话题
- **操作**：直接说"新开会话"
- Agent会做收尾（更新MEMORY + push GitHub + write项目空间），新会话3分钟接上
- **省在哪**：砍掉历史上下文，每轮回到低成本状态

### 2. 一轮说完，别拆成多轮

- ❌ "帮我改下A文件"→ 改了 → "再改下B"→ 改了 → "还有C"…
- ✅ "帮我改A、B、C三个文件，具体是……"
- **省在哪**：每个"再改下"都带着完整历史，一轮说完只有一次大上下文

### 3. 后台长任务用spawn，别在主会话等

- 编译、搜索、批量处理等耗时任务，Agent可以用子session后台跑
- 你继续跟Agent聊别的，不浪费主会话轮次
- **省在哪**：主会话轮次减少，子session上下文独立且短

---

## 第二梯队：日常习惯优化（省15-25%）

### 4. 指令直接，少铺垫

- ❌ "我觉得我们可能需要考虑一下，是不是应该把那个数据库的表结构更新一下"
- ✅ "更新表结构，加3个字段：xxx"
- **省在哪**：你的输入也是token，Agent输出也短

### 5. 减少重复确认

- 小决策（文件名、格式、顺序）信任Agent的默认判断，不对就纠正
- 只在不可逆动作（发邮件、删除、上线）时确认
- **省在哪**：省掉来回确认的2-4轮

### 6. 用定时任务代替手动触发

- 比如worklog，之前手动说"写日志"→ 读历史→写→同步，至少5轮
- 现在23:45自动跑，0轮主会话消耗
- **省在哪**：零主会话成本

---

## 第三梯队：减少返工（省10-15%）

### 7. 铁律防翻车

- 每次违反铁律后的回滚+重做，至少浪费5-8轮
- 沙箱写代码→编译→push，按流程走，一次到位

### 8. 先想清楚再下指令

- "先别急，我想想"比"先做吧，不对再改"省得多
- 需求改来改去的返工成本最高

---

## 一张图总结

| 操作 | 省积分幅度 | 你的成本 |
|------|-----------|---------|
| 主动换会话 | ⭐⭐⭐⭐⭐ | 说一句"新开会话" |
| 一轮说完需求 | ⭐⭐⭐⭐ | 想清楚再发 |
| 长任务后台跑 | ⭐⭐⭐ | 无感 |
| 指令精简 | ⭐⭐ | 稍微克制 |
| 减少确认 | ⭐⭐ | 信任默认 |
| 定时任务 | ⭐⭐ | 一次性配置 |
| 铁律防翻车 | ⭐⭐ | 无感 |

**一句话：会话短、指令精、后台跑、铁律守。**
