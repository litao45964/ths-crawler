## 二、基础设施准备

### 2.1 检查 MySQL 和 Redis 是否运行

```bash
# 检查 Redis（返回 PONG 表示运行中）
redis-cli ping

# 检查 MySQL 连接（返回 1 表示成功）
mysql -u root -p'Litao45964' -e "SELECT 1"
```

### 2.2 如果未运行，启动服务

```bash
# 启动 MySQL（LaunchDaemon 系统自启动，通常开机自动运行）
sudo launchctl load /Library/LaunchDaemons/com.oracle.oss.mysql.mysqld.plist

# 启动 Redis（用户态进程，软链接 /usr/local/bin/redis-server）
nohup /usr/local/bin/redis-server > ~/redis.log 2>&1 &
```

### 2.3 创建数据库

应用首次启动时 Flyway 会自动创建表，但数据库需要提前建好：

```bash
# 创建 ths_crawler 数据库（字符集 utf8mb4）
mysql -u root -p'Litao45964' -e "CREATE DATABASE IF NOT EXISTS ths_crawler DEFAULT CHARSET utf8mb4"
```

### 2.4 重建数据库（清空所有数据，让 Flyway 从零开始）

```bash
# 适用于：迁移脚本修改后需要重新初始化
mysql -u root -p'Litao45964' -e "DROP DATABASE IF EXISTS ths_crawler; CREATE DATABASE ths_crawler DEFAULT CHARSET utf8mb4"
```

## 三、启动后端（Spring Boot，端口 8100）

### 3.1 设置 Java 版本

> ✅ 已持久化到 `~/.zshrc`，新终端自动生效，无需每次手动 export。

```bash
# 查看已配置的 JAVA_HOME（已写入 ~/.zshrc，永久生效）
echo $JAVA_HOME
# 输出：/Library/Java/JavaVirtualMachines/jdk-17.jdk/Contents/Home

# 确认版本（应显示 17.0.x）
java -version
```

### 3.2 启动命令

```bash
# 进入后端项目目录
cd /Users/litao/Documents/devtools/study/ths-crawler/ths-crawler

# 后台启动（日志输出到 /tmp/ths-backend.log）
# --settings 指定 Maven 配置文件（含镜像和本地仓库路径）
export JAVA_HOME=/Library/Java/JavaVirtualMachines/jdk-17.jdk/Contents/Home && \
  mvn spring-boot:run \
    --settings /Users/litao/.m2/settings-public.xml \
    > /tmp/ths-backend.log 2>&1 &
```

> 首次编译会下载依赖，约 30 秒 ~ 2 分钟。后续启动约 2~5 秒。  
> Flyway 会在首次启动时自动执行 `db/migration/` 下的 SQL 脚本建表。

### 3.3 验证后端是否启动成功

```bash
# 方法1：检查启动日志中的关键字
grep "Started ThsCrawlerApplication" /tmp/ths-backend.log

# 方法2：调用 API（返回 JSON 表示成功）
curl -s --max-time 3 "http://localhost:8100/api/industry-flow/latest?topN=1"

# 方法3：检查端口是否在监听
lsof -i :8100 -P | grep LISTEN
```

### 3.4 停止后端

```bash
# 停止 mvn spring-boot:run 启动的进程
pkill -f "spring-boot:run"
```

## 五、幂等启动脚本（推荐）

> 脚本位置：`ths-crawler/scripts/start.sh`  
> 已搭载 Skill 触发：在 ZCode 中直接说 **"启动项目"** / **"重启项目"** / **"停止项目"** 即可自动执行。

### 5.1 用法

```bash
# 幂等启动（已运行的服务自动跳过）
bash /Users/litao/Documents/devtools/study/ths-crawler/ths-crawler/scripts/start.sh

# 重启（先停后启）
bash /Users/litao/Documents/devtools/study/ths-crawler/ths-crawler/scripts/start.sh restart

# 停止所有服务
bash /Users/litao/Documents/devtools/study/ths-crawler/ths-crawler/scripts/start.sh stop

# 查看运行状态
bash /Users/litao/Documents/devtools/study/ths-crawler/ths-crawler/scripts/start.sh status
```

### 5.2 脚本能力

| 特性 | 说明 |
|------|------|
| **幂等** | 后端/前端已在运行则跳过，不会重复启动 |
| **自动修复** | MySQL/Redis 未运行自动尝试启动 |
| **自动建库** | `ths_crawler` 数据库不存在则自动创建 |
| **超时等待** | 后端最多等 90 秒、前端 30 秒，超时报错 |
| **状态展示** | 启动后自动显示各服务状态 + API 冒烟测试 |
| **着色输出** | 绿色 INFO、黄色 WARN、红色 ERROR |

### 5.3 Skill 触发词

在 ZCode 对话中直接说以下任一短语即可自动执行：

| 触发词 | 对应动作 |
|--------|---------|
| `启动项目` / `启动爬虫项目` | `start.sh start` |
| `重启项目` | `start.sh restart` |
| `停止项目` | `start.sh stop` |
| `查看项目状态` | `start.sh status` |

## 六、日志查看常用命令

### 6.1 后端日志

```bash
# 实时查看后端日志（跟踪模式）
tail -f /tmp/ths-backend.log

# 查看最近 50 行
tail -50 /tmp/ths-backend.log

# 搜索错误和异常
grep -i "ERROR\|Exception\|Caused by" /tmp/ths-backend.log

# 搜索启动成功标志
grep "Started\|Tomcat started" /tmp/ths-backend.log

# 查看 Flyway 迁移日志
grep -i "flyway\|Migrating\|Successfully applied\|Migration" /tmp/ths-backend.log

# 查看 SQL 执行日志
grep -i "sql\|jdbc\|mybatis" /tmp/ths-backend.log

# 查看定时任务执行日志
grep -i "scheduler\|cron\|定时" /tmp/ths-backend.log

# 统计各时间段错误数量
grep "ERROR" /tmp/ths-backend.log | cut -d' ' -f1 | sort | uniq -c
```

## 八、常见问题

| 问题 | 原因 | 解决 |
|------|------|------|
| 编译报 `找不到符号: builder()` | JAVA_HOME 未指向 Java 17，Lombok 不兼容 Java 23 | 已持久化到 `~/.zshrc`，新终端 `source ~/.zshrc` 即可 |
| Flyway 迁移报表不存在 | 迁移脚本表名与实体不一致 | 重建数据库让 Flyway 重新执行 |
| MySQL 连接报 `Access denied` | 密码错误或未指定 | 确认密码 `Litao45964`，或检查环境变量 `MYSQL_PASSWORD` |
| Redis 连接失败 | Redis 未启动 | `brew services start redis` |
| 前端页面空白 | 后端未启动或端口不对 | 确认后端 8100 端口正常，Vite 代理配置正确 |
